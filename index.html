// 01 · CONCEPT

바이브코딩이란
무엇인가?

Andrej Karpathy가 제안한 개념으로, 코드를 한 줄씩 직접 작성하는 대신 AI에게 의도를 전달하고 결과물을 받아 진화시키는 개발 철학입니다.

완벽한 구문보다 명확한 비전이 더 중요한 시대. 당신은 설계자가 됩니다.

// 전통적 개발
function fetchData() {
  const res = await fetch(url);
  if (!res.ok) throw new Error(...);
  return res.json();
}
// 모든 줄을 직접 작성
// 바이브코딩
"에러 처리 포함해서 API 호출하는
 함수 만들어줘. 로딩 상태도 관리해야 해.
 TypeScript로, 재사용 가능하게."

// 의도를 전달하면 코드가 생성됨

// 02 · FEATURES

왜 바이브코딩인가

개발의 패러다임을
근본부터 바꿉니다.

⚡
01
극한의 생산성
아이디어에서 프로토타입까지 시간이 10배 단축됩니다. 반복 코드에 시간을 낭비하지 마세요.
🧠
02
비전에 집중
구문보다 아키텍처를. 세부 구현보다 전체 설계를 고민하는 고차원적 사고로 전환됩니다.
🔄
03
빠른 이터레이션
"이 부분 좀 더 세련되게" 같은 피드백으로 코드를 빠르게 다듬습니다. 피드백 루프가 극적으로 빨라집니다.
🌐
04
낮은 진입장벽
기획자, 디자이너, PM도 실제로 동작하는 프로토타입을 직접 만들 수 있습니다. 개발의 민주화.
🎯
05
의도 중심 개발
What이 아닌 Why를 코드로. 목적이 명확하면 AI가 최선의 How를 찾아냅니다.
📈
06
학습 가속
생성된 코드를 분석하며 새로운 패턴을 빠르게 습득합니다. AI가 최고의 페어 프로그래머가 됩니다.
// 03 · PROCESS

어떻게 시작하나요

01
의도를 자연어로 표현하세요
완벽한 기술 용어 없이도 됩니다. "이런 느낌의 버튼, 클릭하면 모달 뜨고, 애니메이션 있었으면 해" — 이 정도면 충분합니다. 의도가 명확할수록 결과가 좋아집니다.
NATURAL LANGUAGE
02
AI와 대화하며 정제하세요
첫 결과물이 완벽하지 않아도 됩니다. "이 부분은 다르게", "색상을 좀 더 진하게", "반응형으로 바꿔줘" — 대화하듯 수정해 나가세요.
ITERATIVE
03
이해하고, 수정하고, 소유하세요
생성된 코드를 블랙박스로 두지 마세요. 핵심 로직을 이해하고, 필요한 부분을 직접 수정하는 능력이 바이브코더의 진정한 경쟁력입니다.
OWNERSHIP
04
검증하고 배포하세요
AI가 생성한 코드도 검토가 필요합니다. 테스트를 작성하고, 엣지 케이스를 확인하고, 프로덕션에 자신 있게 배포하세요.
SHIP IT
